
#  51单片机项目实战笔记：串口通信 + I2C EEPROM存储 + LCD1602显示

## 一、 项目完整总结与运行流程
本项目实现了一个**“基于电脑串口的不定长字符串掉电保存与显示系统”**。
通过电脑串口助手向单片机发送一段英文字符串，单片机接收完毕后，利用 I2C 总线将数据存入 AT24C02（EEPROM）芯片中。无论单片机何时断电重启，都会自动从 EEPROM 中读取上一次保存的数据，并同步显示在电脑串口助手和 LCD1602 屏幕上。

### 核心
1. **上电复位：** 初始化串口和 LCD1602。单片机通过 I2C 协议从 EEPROM 的地址 `0` 开始挨个读取字节。当读到出厂空白符 `0xFF` 或我们人为设置的字符串结束符 `\0` (0x00) 时，停止读取，并将还原出的字符串显示在屏幕和电脑上。
2. **中断接收：** 单片机主程序处于 `while(1)` 空转等待状态。当电脑发来数据时，单片机触发串口接收中断（`interrupt 4`）。每收到一个字符就存入 `RxBuf` 数组中。
3. **结束判定：** 当中断函数检测到接收的字符是回车键（`\n`）时，判定用户发送完毕。此时将回车键替换为 C语言字符串结束符 `\0`，并将完成标志位 `RxFlag` 置 1。
4. **烧录存储：** 主程序检测到 `RxFlag == 1`，启动 `for` 循环，通过 I2C 总线将 `RxBuf` 数组中的字符（包含结尾的 `\0`）逐个写入 AT24C02，完成后清零标志位，准备下一次接收。

## 二、 I2C总线与 EEPROM 的存储

### 1. I2C 总线的“一问一答”通信机制
I2C 是一种只有两根线（SCL时钟线、SDA数据线）的串行总线。单片机（主机）要把数据写进 AT24C02（从机），必须遵循严格的时序：
*   **起始信号 (Start)：** 单片机声明“自己要开始发数据了”
*   **寻址芯片：** 发送 `0xA0`（AT24C02的写设备地址），EEPROM 收到后回复一个 ACK（应答信号，表示“我听到了”）。
*   **指定房间号 (WordAddress)：** 单片机发送要存入的内部地址（比如地址 `0`），EEPROM 再次回复 ACK。
*   **塞入数据 (Data)：** 单片机把 1 个字节的字符发过去，EEPROM 回复 ACK。
*   **停止信号 (Stop)：** 单片机宣布“本次通信结束”。

### 2. 为什么写 EEPROM 必须加 `Delay(5)`（重要）
在主程序的存储循环中，有这样一段代码：
```c
for (i = 0; i <= RxIndex; i++) {
    AT24C02_WriteByte(i, RxBuf[i]);
    Delay(5);  // 5毫秒
}
```
**解释：** 
当 I2C 总线发送完停止信号后，AT24C02 只是把数据接收到了它的“内部缓存”里。接下来，它需要把数据真正烧录进硬件，这个**物理烧录过程大约需要 5 毫秒（内部写周期）**。
在这个写周期内，AT24C02 是**关闭**的，不会响应 I2C 总线上的任何信号。如果此时单片机紧接着发下一个字节，AT24C02 会直接无视，导致**数据丢失**。因此，每写完一个字节，必须让单片机死等 5ms，才能写下一个字节。

### 3. 为什么要存入 `\0` (0x00)？
为了防止“幽灵数据”。
假如第一次存了 `"Apple"`，第二次存了 `"Hi"`。如果不存 `\0`，第二次的 EEPROM 状态会变成 `'H', 'i', 'p', 'l', 'e'`。
如果我们在 `"Hi"` 后面跟着存入一个 `\0`，状态就变成了 `'H', 'i', '\0', 'l', 'e'`。
下次开机读取时，代码读到 `\0` 就会触发 `break` 停止读取，完美屏蔽了后面的旧数据 `'l', 'e'`，保证了数据的纯洁性。

## 三、LCD1602 没法显示中文

在测试阶段我发现，如果在串口助手中发送中文，LCD1602 屏幕上直接显示乱七八糟的字符（可能是日语）。

## 四、 项目经验与避坑指南 (Tips)
1. **中断里的标志位必须手动清零：** 串口接收触发中断后，硬件会将 `RI` 自动置 1，但退出中断前必须写代码 `RI = 0;`，否则单片机会无限卡死在中断里。
2. **串口测试工具的设置：** 验证代码时，在串口助手发送完数据后需**在结尾加回车**。因为本项目的精简逻辑完全依赖 `\n` 来判定用户是否发完了完整的一句话。
3. **用 LCD_Init() 清屏：** 每次写入新数据前，不需要傻傻地用空格去覆盖旧屏幕，直接调用一次 `LCD_Init()` 即可实现清屏并光标归位，代码更简洁。
